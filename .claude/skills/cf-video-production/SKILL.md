# AI 단편 CF 제작 워크플로우 (9:16 세로)

레퍼런스 영상 한 편을 기준으로, **인물·톤·복장이 일관된** 세로 CF를 **컷 단위**로 만들어 러프컷까지 완성하는 검증된 절차다. (요기보 15→20초 CF 제작에서 정립)

# 운영 로직 — 게이트형 파이프라인 (컴펌 중심) ★

이 스킬의 **작업 순서는 반드시 아래 STAGE를 순서대로** 따른다. **각 STAGE 끝에는 사용자 컴펌 게이트**가 있고, **컴펌 없이는 다음 STAGE로 넘어가지 않는다.** (즉흥 생성으로 캐릭터가 흔들리고 컨셉이 바뀌며 낭비되는 것을 원천 차단 — 티렉스 프로젝트에서 도출된 규칙.)

```
STAGE 0 프로젝트 정의   → 컴펌 → STAGE 1 에셋 락 → 컴펌 → STAGE 2 스토리보드 → 컴펌
      → STAGE 3 스틸    → 컴펌 → STAGE 4 영상화  → 컴펌 → STAGE 5 조립/편집 → 컴펌
      → STAGE 6 후처리(선택) → 컴펌 → STAGE 7 배포 → 최종
```

| STAGE | 하는 일 | 산출물 | 컴펌 게이트(다음 진입 조건) |
|---|---|---|---|
| **0 정의** | 목표·제품·비율(9:16)·길이·레퍼런스 확정 | 프로젝트 스펙 | 스펙 OK? |
| **1 에셋 락** ★ | 캐릭터/제품 이미지 **업로드** → 주인공 **선정** → **Element 락**(`<<<id>>>`) | element_id, 제품 media_id | "이 캐릭터·이 토큰으로 확정?" |
| **2 스토리보드** | 컷 리스트(컷당 ~3s)·컨셉·대사 확정 | 컷 리스트 | "이 구성으로 확정?" (확정 후 컨셉 변경은 정식 변경) |
| **3 스틸** | 컷별 스틸 생성(**Element 토큰으로만**), count 2 비교 | 컷별 승인 스틸 | "이 스틸들로 영상 갈까?" |
| **4 영상화** | 승인 스틸만 kling 영상화 · 2D 깨짐/일관성 검증 | 컷별 클립 | "이 클립들로 편집 갈까?" |
| **5 조립** | 트림·전환(크로스페이드/푸시인)·로고 인트로/엔딩 | 러프컷 | "러프컷 OK?" |
| **6 후처리** | (선택) 컬러 통일·자막·음악 | 마스터 | "이대로 마무리?" |
| **7 배포** | 캐시버스트·gitignore 예외·업로드 | 최종본 | — |

## 게이트 불변 규칙 (반드시 지킴)
1. **STAGE 1 선행·완결**: 캐릭터를 **Element로 락하기 전엔 STAGE 3(스틸) 진입 금지.** 색·형태 흔들림의 근본 차단.
2. **비용 게이트**: **스틸 승인 없이 영상(비쌈) 생성 금지.** 스토리보드 확정 없이 스틸 생성 금지.
3. **컨셉 잠금**: STAGE 2 확정 이후 컨셉/제품 변경은 "정식 변경"으로 선언하고, **되돌아간 STAGE의 하위 산출물은 무효화**됨을 사용자에게 명시(예: 제품이 U쿠션→Max로 바뀌면 STAGE 3·4 재작업).
4. **상태 추적**: 각 컷은 `{still, video, status: 대기|승인|완료}`를 갖는다. 진행 시 항상 "현재 STAGE / 다음 게이트"를 사용자에게 알린다.
5. **토큰/크레딧 절약**: 게이트 통과 전 대량 생성 금지. 승인분만 다음 단계.
6. **생성 전 비용 고지 + 컴펌 (필수)**: 이미지·영상을 만들기 **직전에 항상 크레딧 예상 비용을 말하고 사용자 컴펌을 받는다.** 컴펌 없이 생성 금지.
   - 참고 단가: `generate_image`(nano_banana) ≈ **1.5cr/장**, `generate_video`(kling3_0 std 3s) ≈ **4.5cr/컷**, Element 생성 = 무료. (필요 시 `get_cost:true`로 사전 확인)
7. **한 장면씩 순차 (배치 금지)**: 여러 컷을 **한 번에 몰아 생성하지 않는다.** 장면(컷) **하나씩** 생성→확인→다음. 스틸도 컷별로 하나씩, 영상도 컷별로 하나씩. (요기보 20편 CF 포함 전 작업 공통)

> 스킬을 호출해 진행하더라도 **이 게이트 밖으로 벗어나지 않는다.** 매 STAGE 종료 시, 그리고 **매 생성 직전**, 비용을 고지하고 "컴펌?"을 물은 뒤에만 진행한다.

## 0. 시작 전에 확보할 것
- **레퍼런스 영상** (예: `video.mp4`) — 구도·리듬·톤·길이의 기준
- **제품** 정의 (예: 요기보 Pod · 아보카도 그린)
- **대표 모델(탤런트)** — 전 컷 동일 인물을 위한 캐릭터 고정 토큰
  - Higgsfield Element 사용: 프롬프트에 `<<<element-id>>>` 삽입 → 얼굴 락
  - (요기보 모델 A 예시 토큰: `<<<c5c8b066-ca75-40e8-9295-fead4a2d05c9>>>`)
- 작업 도구: Higgsfield MCP(generate_image/generate_video/job_status/media_import_url/balance) + 로컬 ffmpeg + Python(cv2/PIL)

## 1. 레퍼런스 해부 (가장 중요)
- ffmpeg로 레퍼런스를 **0.25~0.5초 간격 프레임**으로 떠서 본다: `ffmpeg -ss T -i ref.mp4 -frames:v 1 f.png`
- 분석할 것: **샷 리듬**(긴 오프닝→빠른 중반→긴 엔딩), **컷 수·길이**, **카메라 구도**(클로즈업/핸드헬드/로우앵글), **톤/컬러**, **전환 방식**.
- 씬 전환 자동 감지: `select='gt(scene,0.25)',showinfo`.
- 이 분석으로 **컷 리스트(스토리보드)**를 짠다. 스토리는 레퍼런스의 감정 아크를 따른다(예: 기상→출근→업무→귀가→휴식→로고).

## 2. 컷별 제작 — "스틸 먼저, 승인 후 애니메이션" (크레딧 절약 핵심)
영상 생성은 비싸다(kling std 3s ≈ 4.5cr). 그래서 **항상 스틸→확인→영상** 순서.
1. **스틸 생성** `generate_image` (모델: `nano_banana_2`)
   - **레퍼런스 스틸을 편집(medias role `image`) + Element 토큰**을 함께 넣으면 얼굴·톤·구도를 유지한 채 한 요소만 바꿀 수 있다 → 일관성의 비결.
   - count 2로 뽑아 비교 후 1개 채택.
2. **스틸 검증**: 다운로드해 montage로 얼굴(모델 일치)·복장·구도 확인. 어긋나면 재생성.
3. **애니메이션** `generate_video` (모델: `kling3_0`, `duration:3, mode:"std", sound:"off"`)
   - 채택 스틸을 `media_import_url`→`start_image`로.
   - 프리셋 추천("IN THE DARK" id `24bae836-2c4a-48e0-89b6-49fcc0b21612`)이 뜨면 `declined_preset_id`로 거절.
   - 출력은 항상 716x1284 · 24fps · ~3.04s · 무음.
4. **모션 검증**: 0%/33%/66%/100% 프레임 strip으로 동작·얼굴 유지 확인.

## 3. 일관성 원칙 (계속 어긋나는 지점)
- **얼굴**: 반드시 Element 토큰. 초기 토큰 없이 만든 스틸은 얼굴이 틀어진다 → 재생성.
- **복장**: 장면군마다 통일(예: 아침=회색 줄무늬 잠옷+차콜 바지 / 출근·회사=네이비 블레이저+흰 카라+포니테일 / 저녁=니트). 한 컷만 튀면(머리 풀림/바지 색 다름) 그 컷만 재생성.
- **자세/해부**: 어색한 부위(꺾인 다리 등)는 "그 부분만 교정, 나머지 동일" 프롬프트로 편집.

## 4. 컬러 그레이딩 — 한 편처럼 묶기
- 컷마다 노출/색이 달라 점프가 생긴다. **LAB 색통계 전이(Reinhard)**로 타깃 톤에 수렴시킨다.
- 타깃: ① 가장 마음에 드는 컷의 LAB, 또는 ② **레퍼런스 영상의 LAB**(레퍼런스 룩 매칭).
- 알파 블렌드 권장: `L:0.5, a:0.6~0.75, b:0.6~0.75` (노출은 절반, 색감은 강하게).
- **주의(함정)**: 수백 프레임을 한 배열로 `np.concatenate`하면 ~1GB 폭주로 값이 깨진다 → **스트리밍 float64 누적**으로 mean/std 계산. cv2의 비디오 시킹도 불안정 → **ffmpeg로 PNG 추출 후 cv2.imread**가 안정적.
- 처리: cv2 프레임별 LAB 변환→mp4v 임시→ffmpeg로 H.264 재인코딩.

## 5. 러프컷 편집 (ffmpeg)
- **비트 단위 조립**: 각 컷을 `-ss/-t`로 트림 + 필요시 `crop` 리프레임 → libx264 crf18 → concat demuxer로 합치고 재인코딩(crf19, +faststart, yuv420p).
- **레퍼런스 리듬 모사**: 컷온액션, ECU 펀치인(crop 확대), 인서트 추가로 템포 조절. 같은 공간 반복은 지루 → **공간·앵글·에너지 다양화**(이동→상호작용→소진 식).
- **리프레임**: 인물이 너무 작거나 머리만 보이면 `crop=w:h:x:y`로 상체가 보이게 재구성(9:16 비율 유지).
- **전환**: 하드컷 기본. 감정 연결은 **푸시인(crop zoom)→다음 컷 디졸브(xfade)** 가 효과적(예: 다이브→잠든 얼굴). `fade=t=out:color=white`로 화이트아웃도 가능(원치 않으면 제거).
- 길이 조정은 각 비트 dur로. **자막을 절대 시간으로 넣었다면, 컷 길이를 바꿀 때 in-point만 바꾸고 dur는 유지**해 자막 타이밍을 지킨다.

## 6. 로고 인트로 / 엔딩
- **엔딩 로고 카드**: 흰 배경 + 공식 로고(투명 PNG 권장, 없으면 흰배경 PNG를 fit) **슬로우 페이드인**(2~3s). `fade=in:alpha` 사용.
- **인트로 로고**: 첫 컷 위에 작게·중앙, 처음부터 떠 있다 다음 컷으로 넘어갈 때 페이드아웃. 흰배경 로고는 near-white→투명 키잉(`alpha=clip((245-min(r,g,b))*k)`)으로 컷아웃해 오버레이.
- **함정**: 이미지 오버레이에 `-loop 1`만 쓰면 무한 스트림으로 멈춘다 → `-loop 1 -t <길이> ... -shortest` 또는 `overlay=...:eof_action=pass`.

## 7. 자막 (CF 문법)
- 하이엔드 CF는 **중간 대사 자막을 최소화**하고 텍스트 무게를 **엔딩 태그라인/로고**에 싣는다. 대사 캡션(예능/유튜브식 하단 pill)은 정보형이라 라이프스타일 CF엔 과함.
- 권장 조합: **에너지 순간=키네틱 키워드**(키워드만 크게+브랜드 컬러 강조) · **감성 순간=에디토리얼 미니멀**(얇은 선 액센트 + 레터스페이스).
- 한글 폰트: 시스템 맑은고딕, 또는 **Pretendard**(OFL, 릴리스 zip의 static ttf). 정밀 모션 타이포는 후보정 몫.
- 자막 버전은 **A(무자막)/B(자막)**로 따로 뽑아 비교 노출.

## 8. 산출물 / 배포
- 러프컷은 무음으로 먼저 확정. **음악은 항상 마지막**(최종 컴펌 후).
- 프론트(Next.js `public/`)에 mp4를 두면 브라우저 캐시 때문에 안 바뀐다 → **URL에 `?v=토큰`** 캐시버스트.
- **Vercel 배포 시 `*.mp4`가 .gitignore면 영상이 빠져 재생 안 됨** → 필요한 러프컷/엔딩 mp4만 `!예외`로 추적해 커밋.
- 사내 공유는 cafe24 FTP 업로드(스토리보드 시트 PNG 등). cafe24는 .mp4 직접서빙이 막혀 있을 수 있음(이미지/시트는 OK).

## 핵심 교훈 (요약)
1. 레퍼런스를 프레임 단위로 해부하고 그 리듬·톤·구도를 따른다.
2. 스틸 먼저 승인받고 애니메이션(크레딧 절약).
3. 얼굴=Element 토큰, 복장·머리 일관성 집착.
4. 레퍼런스 편집+토큰으로 한 요소만 바꿔 일관성 유지.
5. LAB 전이로 전 컷 톤 통일(타깃=최애 컷 또는 레퍼런스).
6. 같은 공간 반복 금지 — 공간·에너지 다양화.
7. 텍스트는 미니멀·엔딩 중심, 음악은 최후.
8. 캐시버스트 + gitignore 예외로 Vercel 재생 보장.
