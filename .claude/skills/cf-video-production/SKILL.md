# AI 단편 CF 제작 워크플로우 (9:16 세로)

레퍼런스 영상 한 편을 기준으로, **인물·톤·복장이 일관된** 세로 CF를 **컷 단위**로 만들어 러프컷까지 완성하는 검증된 절차다. (요기보 15→20초 CF 제작에서 정립)

---

# 운영 로직 — 게이트형 파이프라인 (컴펌 중심) ★

이 스킬의 **작업 순서는 반드시 아래 STAGE를 순서대로** 따른다. **각 STAGE 끝에는 사용자 컴펌 게이트**가 있고, **컴펌 없이는 다음 STAGE로 넘어가지 않는다.** (즉흥 생성으로 캐릭터가 흔들리고 컨셉이 바뀌며 낭비되는 것을 원천 차단 — 티렉스 프로젝트에서 도출된 규칙.)

```
STAGE 0 프로젝트 정의   → 컴펌 → STAGE 1 에셋 락 → 컴펌 → STAGE 2 스토리보드 → 컴펌
      → STAGE 3 스틸    → 컴펌 → STAGE 4 영상화  → 컴펌 → STAGE 5 조립/편집 → 컴펌
      → STAGE 6 후처리(선택) → 컴펌 → STAGE 7 배포 → 최종
```

| STAGE | 하는 일 | 산출물 | 컴펌 게이트(다음 진입 조건) |
|---|---|---|---|
| **0 정의** | 목표·제품·비율(9:16)·길이·레퍼런스 확정 | 프로젝트 스펙 | 스펙 OK? |
| **1 에셋 락** ★ | 캐릭터/제품 이미지 **업로드** → 주인공 **선정** → **Element 락**(`<<<id>>>`) | element_id, 제품 media_id | "이 캐릭터·이 토큰으로 확정?" |
| **2 스토리보드** | 컷 리스트(컷당 ~3s)·컨셉·대사 확정 | 컷 리스트 | "이 구성으로 확정?" (확정 후 컨셉 변경은 정식 변경) |
| **3 스틸** | 컷별 스틸 생성(**Element 토큰으로만**), count 2 비교 | 컷별 승인 스틸 | "이 스틸들로 영상 갈까?" |
| **4 영상화** | 승인 스틸만 kling 영상화 · 2D 깨짐/일관성 검증 | 컷별 클립 | "이 클립들로 편집 갈까?" |
| **5 조립** | 트림·전환(크로스페이드/푸시인)·로고 인트로/엔딩 | 러프컷 | "러프컷 OK?" |
| **6 후처리** | (선택) 컬러 통일·자막·음악 | 마스터 | "이대로 마무리?" |
| **7 배포** | 캐시버스트·gitignore 예외·업로드 | 최종본 | — |

## 게이트 불변 규칙 (반드시 지킴)
1. **STAGE 1 선행·완결**: 캐릭터를 **Element로 락하기 전엔 STAGE 3(스틸) 진입 금지.** 색·형태 흔들림의 근본 차단.
2. **비용 게이트**: **스틸 승인 없이 영상(비쌈) 생성 금지.** 스토리보드 확정 없이 스틸 생성 금지.
3. **컨셉 잠금**: STAGE 2 확정 이후 컨셉/제품 변경은 "정식 변경"으로 선언하고, **되돌아간 STAGE의 하위 산출물은 무효화**됨을 사용자에게 명시(예: 제품이 U쿠션→Max로 바뀌면 STAGE 3·4 재작업).
4. **상태 추적**: 각 컷은 `{still, video, status: 대기|승인|완료}`를 갖는다. 진행 시 항상 "현재 STAGE / 다음 게이트"를 사용자에게 알린다.
5. **토큰/크레딧 절약**: 게이트 통과 전 대량 생성 금지. 승인분만 다음 단계.
6. **생성 전 비용 고지 + 컴펌 (필수)**: 이미지·영상을 만들기 **직전에 항상 크레딧 예상 비용을 말하고 사용자 컴펌을 받는다.** 컴펌 없이 생성 금지.
   - 참고 단가: `generate_image`(nano_banana) ≈ **1.5cr/장**, `generate_video`(kling3_0 std 3s) ≈ **4.5cr/컷**, Element 생성 = 무료. (필요 시 `get_cost:true`로 사전 확인)
7. **한 장면씩 순차 (배치 금지)**: 여러 컷을 **한 번에 몰아 생성하지 않는다.** 장면(컷) **하나씩** 생성→확인→다음. 스틸도 컷별로 하나씩, 영상도 컷별로 하나씩. (요기보 20편 CF 포함 전 작업 공통)

> 스킬을 호출해 진행하더라도 **이 게이트 밖으로 벗어나지 않는다.** 매 STAGE 종료 시, 그리고 **매 생성 직전**, 비용을 고지하고 "컴펌?"을 물은 뒤에만 진행한다.

## 0. 시작 전에 확보할 것
- **레퍼런스 영상** (예: `video.mp4`) — 구도·리듬·톤·길이의 기준
- **제품** 정의 (예: 요기보 Pod · 아보카도 그린)
- **대표 모델(탤런트)** — 전 컷 동일 인물을 위한 캐릭터 고정 토큰
  - Higgsfield Element 사용: 프롬프트에 `<<<element-id>>>` 삽입 → 얼굴 락
  - (요기보 모델 A 예시 토큰: `<<<c5c8b066-ca75-40e8-9295-fead4a2d05c9>>>`)
- 작업 도구: Higgsfield MCP(generate_image/generate_video/job_status/media_import_url/balance) + 로컬 ffmpeg + Python(cv2/PIL)

## 1. 레퍼런스 해부 (가장 중요)
- ffmpeg로 레퍼런스를 **0.25~0.5초 간격 프레임**으로 떠서 본다: `ffmpeg -ss T -i ref.mp4 -frames:v 1 f.png`
- 분석할 것: **샷 리듬**(긴 오프닝→빠른 중반→긴 엔딩), **컷 수·길이**, **카메라 구도**(클로즈업/핸드헬드/로우앵글), **톤/컬러**, **전환 방식**.
- 씬 전환 자동 감지: `select='gt(scene,0.25)',showinfo`.
- 이 분석으로 **컷 리스트(스토리보드)**를 짠다. 스토리는 레퍼런스의 감정 아크를 따른다(예: 기상→출근→업무→귀가→휴식→로고).

## 2. 컷별 제작 — "스틸 먼저, 승인 후 애니메이션" (크레딧 절약 핵심)
영상 생성은 비싸다(kling std 3s ≈ 4.5cr). 그래서 **항상 스틸→확인→영상** 순서.
1. **스틸 생성** `generate_image` (모델: `nano_banana_2`)
   - **레퍼런스 스틸을 편집(medias role `image`) + Element 토큰**을 함께 넣으면 얼굴·톤·구도를 유지한 채 한 요소만 바꿀 수 있다 → 일관성의 비결.
   - count 2로 뽑아 비교 후 1개 채택.
2. **스틸 검증**: 다운로드해 montage로 얼굴(모델 일치)·복장·구도 확인. 어긋나면 재생성.
3. **애니메이션** `generate_video` (모델: `kling3_0`, `duration:3, mode:"std", sound:"off"`)
   - 채택 스틸을 `media_import_url`→`start_image`로.
   - 프리셋 추천("IN THE DARK" id `24bae836-2c4a-48e0-89b6-49fcc0b21612`)이 뜨면 `declined_preset_id`로 거절.
   - 출력은 항상 716x1284 · 24fps · ~3.04s · 무음.
4. **모션 검증**: 0%/33%/66%/100% 프레임 strip으로 동작·얼굴 유지 확인.

## 3. 일관성 원칙 (계속 어긋나는 지점)
- **얼굴**: 반드시 Element 토큰. 초기 토큰 없이 만든 스틸은 얼굴이 틀어진다 → 재생성.
- **복장**: 장면군마다 통일(예: 아침=회색 줄무늬 잠옷+차콜 바지 / 출근·회사=네이비 블레이저+흰 카라+포니테일 / 저녁=니트). 한 컷만 튀면(머리 풀림/바지 색 다름) 그 컷만 재생성.
- **자세/해부**: 어색한 부위(꺾인 다리 등)는 "그 부분만 교정, 나머지 동일" 프롬프트로 편집.

## 4. 컬러 그레이딩 — 한 편처럼 묶기
- 컷마다 노출/색이 달라 점프가 생긴다. **LAB 색통계 전이(Reinhard)**로 타깃 톤에 수렴시킨다.
- 타깃: ① 가장 마음에 드는 컷의 LAB, 또는 ② **레퍼런스 영상의 LAB**(레퍼런스 룩 매칭).
- 알파 블렌드 권장: `L:0.5, a:0.6~0.75, b:0.6~0.75` (노출은 절반, 색감은 강하게).
- **계절/시간대 변주가 의도된 몽타주**(사계절 성장 등)는 타깃을 특정 컷이 아닌 **전 컷 LAB 평균(공통 앵커)**으로 잡고 알파를 낮춰(`L:0.4, a/b:0.55`) 부분 수렴만 — 컷 간 점프는 줄이고 계절 라이트 변주는 보존 (9차 검증: C1↔C2 노출 L 140↔163 점프·C3 앰버 캐스트가 자연스럽게 완화).
- **주의(함정)**: 수백 프레임을 한 배열로 `np.concatenate`하면 ~1GB 폭주로 값이 깨진다 → **스트리밍 float64 누적**으로 mean/std 계산. cv2의 비디오 시킹도 불안정 → **ffmpeg로 PNG 추출 후 cv2.imread**가 안정적.
- 처리: cv2 프레임별 LAB 변환→mp4v 임시→ffmpeg로 H.264 재인코딩.

## 5. 러프컷 편집 (ffmpeg)
- **비트 단위 조립**: 각 컷을 `-ss/-t`로 트림 + 필요시 `crop` 리프레임 → libx264 crf18 → concat demuxer로 합치고 재인코딩(crf19, +faststart, yuv420p).
- **레퍼런스 리듬 모사**: 컷온액션, ECU 펀치인(crop 확대), 인서트 추가로 템포 조절. 같은 공간 반복은 지루 → **공간·앵글·에너지 다양화**(이동→상호작용→소진 식).
- **리프레임**: 인물이 너무 작거나 머리만 보이면 `crop=w:h:x:y`로 상체가 보이게 재구성(9:16 비율 유지).
- **전환**: 하드컷 기본. 감정 연결은 **푸시인(crop zoom)→다음 컷 디졸브(xfade)** 가 효과적(예: 다이브→잠든 얼굴). `fade=t=out:color=white`로 화이트아웃도 가능(원치 않으면 제거).
- 길이 조정은 각 비트 dur로. **자막을 절대 시간으로 넣었다면, 컷 길이를 바꿀 때 in-point만 바꾸고 dur는 유지**해 자막 타이밍을 지킨다.

## 6. 로고 인트로 / 엔딩
- **엔딩 로고 카드**: 흰 배경 + 공식 로고(투명 PNG 권장, 없으면 흰배경 PNG를 fit) **슬로우 페이드인**(2~3s). `fade=in:alpha` 사용.
- **인트로 로고**: 첫 컷 위에 작게·중앙, 처음부터 떠 있다 다음 컷으로 넘어갈 때 페이드아웃. 흰배경 로고는 near-white→투명 키잉(`alpha=clip((245-min(r,g,b))*k)`)으로 컷아웃해 오버레이.
- **함정**: 이미지 오버레이에 `-loop 1`만 쓰면 무한 스트림으로 멈춘다 → `-loop 1 -t <길이> ... -shortest` 또는 `overlay=...:eof_action=pass`.

## 7. 자막 (CF 문법)
- 하이엔드 CF는 **중간 대사 자막을 최소화**하고 텍스트 무게를 **엔딩 태그라인/로고**에 싣는다. 대사 캡션(예능/유튜브식 하단 pill)은 정보형이라 라이프스타일 CF엔 과함.
- 권장 조합: **에너지 순간=키네틱 키워드**(키워드만 크게+브랜드 컬러 강조) · **감성 순간=에디토리얼 미니멀**(얇은 선 액센트 + 레터스페이스).
- 한글 폰트: 시스템 맑은고딕, 또는 **Pretendard**(OFL, 릴리스 zip의 static ttf). 정밀 모션 타이포는 후보정 몫.
- 자막 버전은 **A(무자막)/B(자막)**로 따로 뽑아 비교 노출.

## 8. 산출물 / 배포
- 러프컷은 무음으로 먼저 확정. **음악은 항상 마지막**(최종 컴펌 후).
- 프론트(Next.js `public/`)에 mp4를 두면 브라우저 캐시 때문에 안 바뀐다 → **URL에 `?v=토큰`** 캐시버스트.
- **Vercel 배포 시 `*.mp4`가 .gitignore면 영상이 빠져 재생 안 됨** → 필요한 러프컷/엔딩 mp4만 `!예외`로 추적해 커밋.
- 사내 공유는 cafe24 FTP 업로드(스토리보드 시트 PNG 등). cafe24는 .mp4 직접서빙이 막혀 있을 수 있음(이미지/시트는 OK).

## 9. 리얼 다큐 톤 락 — "AI 티" 제거 (요기보 10주년서 도출) ★
생성물이 "AI스러워" 보이는 건 대부분 **연출 톤**의 문제다. 레퍼런스가 실사 다큐면, 그 물리·조명·카메라 문법을 그대로 모방한다.
- **제품은 의인화해 걷게 하지 말 것.** 실제 CF의 제품(빈백)은 **직립으로 서서 젠틀 스웨이**(천 흔들림·미세 기우뚱)하거나, 원하면 **통통 스프링 바운스**(스쿼시&스트레치→형태 리커버리)로만 움직인다. "앞으로 걷는 bobbing walk"는 즉시 가짜 티.
- **카메라 = 로우앵글 + 락(고정)/초저속 푸시인/팔로우.** 올려다보는 앵글로 제품을 주인공화. CG처럼 휙 도는 스윙·플래시 남발 금지.
- **조명은 실제 환경광**(자연광/저녁 듀스크/실내 실사)으로. 검은 보이드 + 골드 스포트라이트 남발은 AI 티의 주범.
- **배경에 실제 사람·환경**(흐린 실사 하객 실루엣·보케)을 깔아 그라운딩. **저채도 필믹 그레이드 위에 제품 컬러만 톡** 튀게.
- 절차: 레퍼런스의 해당 구간을 **0.25초 단위로 뜯어** 카메라 각도·무빙·제품 모션을 표로 정리한 뒤, 그 문법을 프롬프트에 그대로 박는다.

### 룩 락 (측정 기반) — 화면 온도·톤을 전 컷 통일 ★
"화면 온도(색온도)·그레이드"는 Element로 못 박는다(Element=피사체 전용). 대신 **레퍼런스 프레임을 cv2로 측정해 수치로 룩을 정의**하고 3겹으로 고정한다.
- **측정**: 프레임들의 평균 RGB(웜지수 R/B), LAB 평균(a·b 색편향), 블랙레벨(5%ile), 하이라이트(95%ile)를 뽑아 색온도·매트감·대비를 수치화. (예: 요기보 가족편 = R/B 1.11 웜 · LAB a131 b132 살구빛 · 블랙 35(리프티드) · 하이라이트 211(소프트) → "따뜻·필름·소프트 대비·얕은 심도")
- **3겹 적용**: ①프롬프트에 룩 문구 삽입 ②**컷별 원본 프레임을 이미지 레퍼런스로 투입**(색온도·프레임 자동 상속, 최대 레버) ③전 컷 생성 후 **LAB 전이/LUT 그레이드 한 번에 적용**해 색온도·블랙·하이라이트 완전 통일(무료 ffmpeg, §4). ③이 최종 통일자.
- "미리 룩을 element로" 요구 시: Higgsfield엔 룩 Element 없음을 알리고, **스타일 앵커 프레임**(대표/그레이드 플레이트)을 스타일 레퍼런스로 재사용하는 방식으로 대체.

## 10. 확장 파이프라인 (OpenMontage 흡수 — 필요 시 도입)
우리 엔진(Higgsfield MCP + ffmpeg + React 프론트)에 접목 가치가 높은 외부 기법. **설치보다 기법 흡수**를 기본으로 한다(외부 스킬은 자동 실행 대상이라 검토 없이 설치 금지).

- **레퍼런스 해부 자동화** (`video-understand` 계열): ffmpeg **씬 자동감지 + 키프레임 추출 + Whisper 자막 전사**를 한 스크립트로 → JSON. API 키 불필요(로컬 ffmpeg+`openai-whisper`). STAGE 1 해부를 표준화. (`select='gt(scene,0.3)'` + `-vf fps` + whisper)
- **Remotion 타이포/자막/엔딩 레이어** (React): 러프컷(Higgsfield+ffmpeg) 위에 **자막·로고 인트로/엔딩·"숫자" 리빌·데이터비주얼·오디오 파형**을 코드로 렌더. 우리 프론트가 Next.js/React라 궁합 최상. drawtext 대비 재현성·온브랜드↑. (STAGE 6/7)
- **오디오 마감** (`elevenlabs`: TTS/BGM/SFX): 러프컷 확정 후 **한국어 내레이션(멀티링구얼)·BGM 생성·텍스트→효과음**. "음악은 최후" 원칙의 실행 도구. `ELEVENLABS_API_KEY` 필요.
- **생성 엔진 대안 — Seedance 2.0** (`ai-video-gen`, fal.ai `FAL_KEY`): **단일 패스 네이티브 오디오 + 멀티샷 + 감독급 카메라 컨트롤 + 립싱크**. 카메라 무빙/멀티샷이 중요한 컷은 Higgsfield Kling과 **비교 테스트** 후 채택. (`ltx2`=Modal 기반 I2V 대안)
- **다국어 버전** (`video-translate`): 확정본을 글로벌 더빙 버전으로. (아바타·토킹헤드 계열 create-video/heygen/avatar-video/faceswap = 제품 CF엔 적합도 낮음.)

## 11. 제품 데이터 레지스트리 — Element 락·사이즈 정확도의 단일 소스 ★

제품이 등장하는 모든 생성 전에 **`/products` 페이지(= Mongo `products` 컬렉션, `GET /api/products`)를 먼저 조회**한다. 가족 CF에서 도출·검증된 체계.

- **구조**: 제품별 `{ name, category, colors[{color, hex, sprite360(cafe24 360 스프라이트 URL), elementId, elementName}], spec{w,h,d,weight}, scalePrompt, notes, usedIn }`
- **Element 락 절차**: ① 레지스트리에서 해당 색상의 `elementId` 확인 → 있으면 그대로 `<<<id>>>` 사용. ② 없으면 `sprite360` URL 다운로드 → 프레임 분할(가로 스프라이트, 폭÷높이=프레임 수) → **정면/측면/뒤 3각도** 선별 → media_upload → Element 생성(무료) → **elementId를 레지스트리에 기록**(PUT `/api/products/:id`). 색상이 다르면 HSV 리컬러 후 락(문필로우 사례: 블루→올리브, H=38).
- **사이즈 정확도**: 생성 모델은 cm 숫자를 못 읽는다 → `spec`은 **`scalePrompt`(인물 대비 비교 언어)**로 변환해 프롬프트에 주입. 예: "Max(170cm) = a bean bag long enough for an adult to lie on". 생성 후엔 프레임에서 인물 대비 비율을 spec과 대조 검수.
- **워크스페이스 주의**: Higgsfield Element는 삭제될 수 있음(실제 사고: yogibo-wife-real 소실) → 레지스트리의 elementId가 죽어 있으면 `show_reference_elements(list)`로 확인 후 재락하고 ID 갱신. **레지스트리가 원본, Element는 캐시**라는 관점.

## 12. 엔진 선택 & 원테이크 무드필름 (7·8차 파티/스위트에서 도출 · 2026-07-09) ★

**엔진은 "무엇이 주인공인가"로 고른다:**
- **카메라 안무·원테이크가 주인공** (공간 리빌·트래킹·통과 샷 중심 무드필름) → **Seedance 2.0**. 유일하게 `video_references` 입력을 받는다 — 마케터/레퍼런스 영상을 넣으면 **카메라 동선·페이싱을 그대로 따라간다** (텍스트로 비는 게 아니라 동선 자체를 락).
- **인물 마이크로 모션이 주인공** (토닥임·미소·뒤척임 등 3s 단일 컷) → **Kling 3.0** 3s pro가 더 싸고 충분. 큰 카메라 안무는 텍스트만으로 재현 리스크.

**원테이크 레시피 (검증)**: `start_image`(승인 스틸) + `video_references`(동선 레퍼런스) + `image_references`(미술 키비주얼 — 케이크·로고 등) + **타임라인식 프롬프트**("0-2.5s: …, 2.5-4.5s: …") + 레퍼런스 초월 지시("smoother, professionally eased, speed-ramp through the window"). 네이티브 오디오는 **무료**(가격 동일)이고 화면 싱크 좋음 — 오디오 아크(정적→스웰→클라이맥스)까지 프롬프트에 설계.

**비용 운영**: 어떤 모델이든 `get_cost:true` 프리플라이트로 **생성 전 정확한 크레딧 무료 조회** — 추정으로 견적 내지 말 것. 실측(2026-07): Seedance std 720p 4.5cr/s · 1080p 9cr/s · 오디오 무료 / Kling pro 오디오on 2.5cr/s · 무음 1.75cr/s. 운영은 720p(또는 무음) 검증 1회 → 통과 시 1080p(오디오) 마스터.

**군중/파티 씬 = 베이스 프레임 + 부분 얼굴 교체**: 여러 인물 밀집 구도를 제약(특정 인물·정면 금지·제품·통로) 걸고 맨땅 조립하면 반드시 어색해진다(4명이 띄엄띄엄, 합성 티). → 레퍼런스 영상의 해당 프레임을 베이스로 넣고 **주요 2~3명만 `<<<element>>>` 얼굴 교체, 나머지 군중은 그대로**. 단 **Element 사용 시 이미지 참조는 2장 이하** — 3장+Element 3인 조합은 잡이 10분 걸리다 failed (과금은 안 됨).

**합성 티(콜라주 느낌) 제거 2단 콤보**: ① 통합 그라운딩 편집 1회 — "구도·오브젝트 전부 유지, 조명·접지만": 웜 광원 통일 + 모든 오브젝트 접촉 그림자 + 카펫 눌림 + 창가 림라이트 (참조 락된 오브젝트가 각자 원본 스튜디오 조명을 유지해 스티커처럼 뜨는 근본 원인 해결) ② 무료 필름 그레이드(그레인·할레이션·비네트). 케이크 등 키비주얼은 "정확히 동일 유지" 지시가 원본 조명까지 복사하니 **정체성 유지와 조명 적응 지시를 분리**할 것.

**Gemini 워터마크**: nano banana 계열 스틸엔 스파클(✦) 워터마크가 박힘 — 위치 고정이라 cv2 인페인팅(영역 통짜 마스크)으로 무료 제거. start_image로 쓰기 전에 제거. 스파클은 4개 로브라 로브별 작은 마스크로는 중심이 남음 — 한 덩어리 사각 마스크로.

**포즈 뼈대(스캐폴드) 워크플로우 (9차에서 표준화 · 2026-07-09)**: 인물 포즈가 중요한 스틸은 생성 전에 **무료 뼈대 사진을 먼저 확정**한다 — 소스: ①무료 스톡(Pexels·Unsplash·Pixabay, 상업 무료) ②포즈 마네킹 툴(PoseMy.Art·JustSketchMe 무료 티어) 스크린샷 ③셀프 촬영. 확정된 뼈대 사진을 base-frame으로 주입하고 정체성·제품·미술만 교체. ControlNet식 스켈레톤 맵은 힉스필드 모델이 입력받지 못하므로 **실사 포즈 사진이 뼈대의 정답**. 사용자와 뼈대를 먼저 컨펌하면 "생성 후 구도 불만 → 재생성" 루프가 끊긴다. 영상 모션 뼈대는 motion_control(퍼펫티어) 파일럿 후보 — 사용 전 get_cost 실측.

**빈백 형태 락**: 프롬프트에 "sofa"라는 단어를 쓰면 팔걸이·등받이 소파로 드리프트한다 → "NO armrests, NO backrest, simple rounded pill shape" + 레지스트리 360 단독 프레임을 형태 참조로 동봉. 아무도 앉지 않은 팟/빈백은 "plump, smooth undented top" 명시 (인형을 지우면 눌린 자국이 남는 사고 있음).

## 13. 프로젝트 운영 리추얼 — 사용자 표준 진행 방식 (반드시 이 순서로 리드) ★★

사용자가 매번 다시 설명하지 않도록, **새 프로젝트가 시작되면 Claude가 먼저 이 순서를 제시하고 리드한다** ("지난 프로젝트들도 이렇게 갔습니다 — 다음 게이트는 OO입니다"):

1. **스토리보드 페이지부터** — `src/app/storyboardN/page.js` 생성: 게이트 파이프라인 + 섹션/컷보드 + 에셋 락 현황 + 산출물 슬롯 + 비용 플랜(실측 단가). 홈(page.js)에 프로젝트 카드 추가. dev 서버 200 확인.
2. **에셋 락** — 레지스트리(/products) 조회 → Element·360·승인 스틸·키비주얼·포즈 뼈대 사진(§12) 확보. 전부 무료 준비 작업.
3. **컷별 스틸 루프** — 컷마다: 뼈대/베이스 확정(사용자 컨펌) → "OO 스틸 · Xcr 사용할까?" 승인 → 생성 1장 → 검수 소견과 함께 제시 → 수정 요청 반영(최소 편집 원칙) → **승인 즉시 public/에 복사하고 스토리보드 페이지 산출물 슬롯에 게시 + 컷보드 상태(⬜→🟡→✅) 갱신**.
4. **컷별 영상화 루프** — 전 컷(또는 해당 컷) 스틸 승인 후: "OO 영상화 · Xcr(실측) 사용할까?" 승인 → 생성 → 프레임 그리드로 검수 → 페이지 게시.
5. **러프컷 조립 (무료 · 생략 금지)** — 컷이 2개 이상 모이면 ffmpeg로 러프컷을 만들어 **페이지에 러프컷 플레이어로 게시**하고 흐름을 검수받는다. 컷 추가·수정 때마다 러프컷 버전업(v1, v2…). 최종 컴펌도 러프컷 위에서.
6. **후반 (무료)** — 톤 통일(LAB) → 태그라인/로고 → 음원(항상 마지막, Pixabay/네이티브 오디오) → 캐시버스트 배포.

**불변 규칙 (전 단계 공통)**:
- 크레딧 쓰는 모든 호출은 직전에 "OO작업 · Xcr 사용할까?" → **명시적 OK 후에만** 실행. 방향 지시("해줘봐")는 지출 승인이 아님. 이미지 1장씩.
- **모든 크레딧 산출물은 즉시 프론트(스토리보드 페이지)에 게시** — 채팅에만 보여주고 끝내지 않는다.
- 단가는 get_cost 프리플라이트 실측만 (추정 견적 금지). 지출 누계·잔액을 보고에 포함.
- **새로운 과정/기법이 검증되면 그 자리에서 이 SKILL.md와 메모리에 추가** — 사용자가 시키기 전에.
